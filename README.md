# 🎭 Character Chat

A small-scale language model demo built with Node.js + Express and the Gemini 2.5 API. Type any character name - fictional, historical, or literary - and have a real conversation with an AI imitation of them.

## Features

- Chat with any character (Sherlock Holmes, Yoda, Socrates, Hermione Granger…)
- Conversation history maintained across messages
- Family-friendly safety guardrails baked into the system prompt
- Rate limiting (30 messages/minute per IP)
- API key stored securely on the server — never exposed to the browser
- Input validation and XSS prevention throughout

## Security design

| Threat | Mitigation |
|---|---|
| API key exposure | Key lives in `.env` on the server only; never sent to the client |
| XSS | All user input rendered via `textContent`, never `innerHTML` |
| Oversized payloads | `express.json({ limit: '16kb' })` |
| Prompt injection / abuse | Input sanitized and capped (100 chars for character, 500 for messages) |
| Abuse / cost runaway | Rate limiter: 30 requests/minute per IP |
| CORS abuse | `cors({ origin: false })` — only same-origin requests reach `/api/` |
| History manipulation | Message count capped at 40; each message validated for role + length |

## Prerequisites

- [Node.js 18+](https://nodejs.org/)
- An [Anthropic API key](https://console.anthropic.com/)

## Setup

```bash
# 1. Clone or download this repo
git clone https://github.com/irstotts/character-chat.git
cd character-chat

# 2. Install dependencies
npm install

# 3. Create your environment file
cp .env.example .env

# 4. Open .env and paste your key - instructions are listed in the file.

# 5. Start the server
npm start
```

Then open **http://localhost:3000** in your browser.

For development with auto-restart on file changes:
```bash
npm run dev
```

## How it works (AI concepts)

This project demonstrates several core language model concepts:

1. **System prompts** — A hidden instruction tells the model to stay in character, setting its persona before the conversation begins.

2. **Conversation history** — Every turn, the full `[{role, content}]` array is sent to the API. The model has no built-in memory; the client accumulates history and re-sends it each time.

3. **Role-based messaging** — The API distinguishes `user` and `assistant` turns, letting the model understand the conversational back-and-forth.

4. **Safety via prompting** — Guardrails (family-friendly, no harmful content) are enforced through explicit system prompt instructions.

5. **Token budgeting** — `max_tokens: 300` caps response length, keeping replies conversational and controlling API cost.
